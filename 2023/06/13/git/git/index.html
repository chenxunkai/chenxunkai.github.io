回滚代码–git reset –hard/soft/mixed

  • reset参数是重置命令
  • hard是重置代码仓库版本

回滚模式有三种：

  • –soft,头指针恢复，已经add的暂存区以及工作空间的所有东西都不变
  • –mixed,头指针恢复，已经add的暂存区会丢失，工作空间代码不变
  • –hard,头指针恢复，已经add的暂存区会丢失，工作空间代码恢复

此外，还有两种回滚方式：HEAD是指向当前仓库

  • 回滚到master上一次提交的版本，其中：^代表上一个版本的意思，HEAD指向当前仓库
  • 回滚到当前仓库的指定版本,其中：~3，代表以当前版本为基数，回滚3次。即HEAD~3代表回滚至master前三个版本

查看提交后文件是否改动–git status

若提交后文件未改动，输入git status命令后将显示

此时新增文件test.txt,则会在编辑器该文件右侧显示U，代表未提交状态,此时输入git status命令查看当前状态

修改test.txt内容,则会在编辑器该文件右侧显示M，代表修改后未提交状态,此时输入git status命令查看当前状态

此时提交到本地仓库缓存区git add .(. 表示全部文件)，此时编辑器中新增的那个文件右侧从U显示为A，代表新添加的文件,输入git status命令查看当前状态

此时正式提交到本地仓库git commit -m 'add new aaa.txt / modify file test.txt',发现编辑器各文件右侧字母已消失,输入git status命令查看当前状态工作区是干净的nothing to commit, working tree clean

vscode编辑器文件右侧字母解释：
U - Untracked
表示该文件是未被Git跟踪的文件，Git不会自动将其包含在版本控制中。这意味着该文件不会被提交到版本库中，也不会被包含在Git的快照中。如果希望Git开始跟踪该文件，需要使用git add命令将其添加到暂存区，然后文件的状态会从U（Untracked）变为A（Added）。
A - Added
表示该文件是新添加的文件，已经被Git跟踪，并且将会包含在下一次的提交中。当使用git add命令将新文件添加到暂存区后，文件的状态会从U（Untracked）变为A（Added）。
M Modified
表示该文件已被修改。当对已跟踪的文件进行了修改后，文件的状态会从A（Added）变为M（Modified）。这意味着该文件在上一次提交之后发生了变化，但尚未被添加到暂存区

合并分支–git merge 分支名

在开发中,建议无论做什么操作都使用分支,拉取时master后创建分支，分支名改为你要做的操作，一般情况下，拉取master后，想要修改功能或者添加功能，都是创建分支，在分支里修改不影响master，如果修改错了代码或者误删之类的,从master上拉取一份就行。

创建并切换分支git checkout -b dev2,新建a.txt,输入内容,然后提交（依次输入git add .,git commit -m '创建dev分支新建a.txt'）
切换分支至master，此时a.txt在工作区消失，此文件在dev分支上，现在将dev内容合并至master分支上git merge dev

保存当前工作状态–git stash

查看当前存储了多少工作状态–git stash list

状态工作恢复–git stash pop

切换分支至dev，修改a.txt内容，此时若切换分支git checkout master,会报错，因为你在分支上修改文件或者其它功能时，在没有提交到本地仓库前，想要切换或者创建到其它分区是会报错的

此时可使用git stash保存当前工作状态,切换至其他分支完成其他功能，也可以使用git stash list查看当前存储了多少工作状态，完成其他分支功能后可切换回原来的分支使用git stash pop恢复工作状态
