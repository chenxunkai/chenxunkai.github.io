搭建GitHub+picgo图床

GitHub

  • 创建github新仓库，new repository

  • 创建后依据提示在本地建立连接远程库

  • 创建token令牌，setting -> Developer settings -> Personal access tokens

  • 设置好后，点击Generate token生成token，token待会用于配置picgo

PicGo

  • 到github上下载picgo：Molunerfinn/PicGo at v2.3.1 (github.com)

  • 下载安装后，打开该软件时是不会跳出界面的，默认显示在右下角的任务栏上

  • 到插件设置上下载github-plus插件，用于同步上传删除github上的图片

  • 配置githubPlus

设置了自定义域名后，需在hosts(C:\Windows\System32\drivers\etc)上最后一行添加 104.16.85.20 cdn.jsdelivr.net

  • 配置完成

# Typora

  • 使用Typora上传图片时设置：格式 -> 图像 -> 全局图像设置

上传错误

若picgo出现上传错误，可从picgo设置 -> 设置日志文件 -> 打开日志文件中查看具体错误信息

  • unable to verify the first certificate

在日志上若出现此情况，则应该是使用github时需开启steam++来加速github，此时只要关闭加速即可。

  • 上传时出现 StatusCodeError: 422 的原因

因picgo目前不支持上传同名文件到github上，所以上传同名时会报错，解决方案：可在Picgo 开启上传时重命名和时间戳重命名。
